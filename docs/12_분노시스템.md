# 분노(Rage) 시스템

---

## 개요

분노 시스템은 **스킬 시스템 위에 구축**된다. 별도의 하드코딩된 분노 로직 없이, 빌트인 스킬 3종(일반 공격 → 분노 축적 → 분노 공격)의 조합으로 동작한다. 플레이어와 적 모두 동일한 로직(`processUnitTurn` / `ProcessUnitTurn`)을 사용한다.

---

## 핵심 구조

```
일반 공격(ilban_attack)
  ├── 물리 공격 (계수: 1.0)
  └── TRIGGER_SKILL → rage_accumulate
                        └── ADD_RAGE (useSourceStat: true → unit.ragePerAttack 사용)

rage >= maxRage일 때:
  분노 공격(bunno_attack) 자동 발동
  ├── CONSUME_RAGE (100)
  └── 물리 공격 (계수: 티어별 상이)
```

| 항목 | 값 |
|------|-----|
| Hierarchy | BUILTIN |
| Tags | RAGE, PHYSICAL |
| Trigger | 매 턴 + RAGE_FULL 조건 |
| Effect 1 | CONSUME_RAGE (100) |
| Effect 2 | ATTACK — PHYSICAL, coefficient: 데이터 테이블 참조 |

### rage_accumulate (분노 축적)

| 항목 | 값 |
|------|-----|
| Hierarchy | LOWEST |
| Tags | RAGE |
| Trigger | ON_SKILL_ACTIVATION(ilban_attack) |
| Effect | ADD_RAGE — amount: 데이터 테이블 참조, **useSourceStat: true** |

## useSourceStat 메커니즘

`ADD_RAGE` 이펙트에 `useSourceStat: true` 플래그가 설정되면, 이펙트의 `amount` 대신 **유닛의 `ragePerAttack` 스탯**을 사용한다.

### ragePerAttack 결정 방식

- **플레이어**: `BattleUnit` 생성 시 `extractRagePerAttack()`가 `rage_accumulate` 스킬의 `ADD_RAGE` effect에서 `amount` 값을 추출
- **적**: `EnemyTemplate`의 `ragePerAttack` 필드 (enemy.data.json에서 로드, 기본값 25)

```
// TypeScript
private extractRagePerAttack(): number {
  for (const skill of this.activeSkills) {
    if (skill.id === 'rage_accumulate') {
      for (const eff of skill.effects) {
        if (eff.type === SkillEffectType.ADD_RAGE) return eff.amount;
      }
    }
  }
  return 0;
}

// C#
private int ExtractRagePerAttack() {
  foreach (var skill in ActiveSkills) {
    if (skill.Id == "rage_accumulate") {
      foreach (var eff in skill.Effects) {
        if (eff.Type == SkillEffectType.ADD_RAGE) return eff.Amount;
      }
    }
  }
  return 0;
}
```

### SkillExecutionEngine에서의 처리

```
// TypeScript
case SkillEffectType.ADD_RAGE: {
  const rageAmount = effect.useSourceStat ? source.ragePerAttack : effect.amount;
  source.rage += rageAmount;
  ...
}

// C#
case SkillEffectType.ADD_RAGE: {
  int rageAmount = effect.UseSourceStat ? source.RagePerAttack : effect.Amount;
  source.Rage += rageAmount;
  ...
}
```

## 전투 턴 처리 흐름 (processUnitTurn / ProcessUnitTurn)

플레이어와 적 모두 동일한 로직을 사용한다:

```
1. 스턴 체크 → 스턴 중이면 턴 스킵

2. 메인 스킬 결정
   - bunno_attack 트리거 조건 충족(RAGE_FULL)? → 분노 공격
   - 아니면 → 일반 공격(ilban_attack)
   - 둘 다 없으면 → 기본 공격(ProcessBasicAttack)

3. 메인 스킬 실행 → 로그 + 흡혈 적용

4. UPPER 스킬 실행 (트리거 조건에 따라)

5. 멀티히트 체크 → 추가 타격

6. 분노 게이지 만충 체크 (메인이 일반 공격이었을 때)
   - rage >= maxRage → bunno_attack 추가 발동

7. 반격 체크
```

### 적의 분노 처리 (리팩토링 전 vs 후)

**리팩토링 전** — 하드코딩:
```
// ProcessEnemyTurn에서 직접 처리
enemy.Rage += BattleDataTable.Rage.PlayerRagePerAttack;
if (enemy.Rage >= enemy.MaxRage) {
  enemy.Rage = 0;
  int rageDamage = (int)(enemy.GetEffectiveAtk() * BattleDataTable.Rage.AttackMultiplier);
  target.TakeDamage(rageDamage);
}
```

**리팩토링 후** — 스킬 시스템:
```
// ProcessUnitTurn (플레이어/적 동일)
// 적에게 빌트인 스킬이 부여되어 있으므로 동일한 스킬 체인 동작:
// ilban_attack → rage_accumulate(ADD_RAGE) → bunno_attack(CONSUME_RAGE + ATTACK)
```

## rage_mastery (분노 마스터리)

### 리팩토링 전 — STAT_MODIFIER

```
// PassiveType.STAT_MODIFIER, 특수 처리
ragePowerMultiplier += value;
// bunno_attack 데미지에 곱셈 적용
damage = (int)(damage * unit.ragePowerMultiplier);
```

### 리팩토링 후 — SKILL_MODIFIER

```
// PassiveType.SKILL_MODIFIER, targetTag: SkillTag.RAGE
// 기존 skillTagBonuses 시스템 활용
skillTagBonuses[RAGE] += value;
// 모든 RAGE 태그 스킬에 자동 적용 (bunno_attack, bunno_thunder 등)
getSkillDamageMultiplier(tags) → 1 + sum(bonuses)
```

## 적 생성 시 스킬 부여

### EnemyTemplate.buildEnemySkills / BuildEnemySkills

```
// TypeScript
private buildEnemySkills(): ActiveSkill[] {
  const builtins = ActiveSkillRegistry.getBuiltinSkills();
  const rageAccum = ActiveSkillRegistry.getById('rage_accumulate', 1);
  const skills = [...builtins];
  if (rageAccum) skills.push(rageAccum);
  skills.push(...this.activeSkills);
  return skills;
}

// C#
private ActiveSkill[] BuildEnemySkills() {
  var builtins = ActiveSkillRegistry.GetBuiltinSkills();
  var rageAccum = ActiveSkillRegistry.GetById("rage_accumulate", 1);
  var skills = new List<ActiveSkill>(builtins);
  if (rageAccum != null) skills.Add(rageAccum);
  skills.AddRange(ActiveSkills);
  return skills.ToArray();
}
```

적 인스턴스 생성 시:
- 빌트인 스킬 (ilban_attack, bunno_attack) + rage_accumulate(티어1) + 적 고유 스킬
- `unit.ragePerAttack = this.ragePerAttack` (enemy.data.json의 값)

## 제거된 항목

| 항목 | 이전 위치 | 비고 |
|------|-----------|------|
| `ragePowerMultiplier` | BattleUnit | rage_mastery가 SKILL_MODIFIER로 전환됨 |
| `playerRagePerAttack` | battle.data.json → RageConfig | rage_accumulate 티어 데이터로 이동 |
| `attackMultiplier` | battle.data.json → RageConfig | bunno_attack 계수 데이터로 이동 |
| `SkillConfig` (전체) | BattleDataTable | 사용처 없음 |
| `ProcessEnemyTurn` | Battle.cs | ProcessUnitTurn으로 통합 |
| `ProcessPlayerTurn` | Battle.cs | ProcessUnitTurn으로 통합 |
| `RAGE_POWER` StatType 사용 | BattleUnit | enum 정의는 호환성 위해 유지, 사용처 없음 |
