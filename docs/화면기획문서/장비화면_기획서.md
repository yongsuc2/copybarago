# 장비 화면 기획서

---

## 1. 화면 구성 (위 → 아래)

### 1-1. 화면 헤더
```
장비
```

### 1-2. 탭 바
```
[ 장비 ] [ 합성 (N) ]
```
- 2탭 구성: 장비 탭 + 합성 탭
- 합성: 합성 가능 그룹 수 표시
- 활성 탭은 강조 스타일 적용
- 기본 탭: 장비

---

## 2. 탭별 화면 구성

### 2-1. 장비 탭 (페이퍼돌 + 보관함)

캐릭터를 중앙에 배치하고, 양옆으로 장비 슬롯을 RPG 페이퍼돌 형태로 표시한다.
하단에 보관함 장비 아이콘 그리드를 배치한다.

#### 페이퍼돌 레이아웃 (4행 × 3열 CSS Grid)
```
┌────────┬────────┬────────┐
│ 목걸이  │  모자   │  무기   │
├────────┼────────┼────────┤
│ 반지1   │        │ 방어구  │
├────────┤ 캐릭터  ├────────┤
│ 반지2   │        │  장갑   │
├────────┼────────┼────────┤
│        │  신발   │        │
└────────┴────────┴────────┘
```
- 중앙열 2-3행: CharacterSprite (캐릭터, row span 2)
- 각 슬롯 셀: 56×56px

#### 화면 배치 순서 (위 → 아래)
1. 페이퍼돌 (4행×3열)
2. 장비 상세 패널 (슬롯/보관함 클릭 시에만 표시)
3. 플레이어 총 능력치 요약 (ATK / HP / DEF / CRIT)
4. 보관함 헤더 + 필터 탭 바 + 아이콘 그리드

#### 슬롯 셀 디자인

**빈 슬롯:**
- `#1a1a2e` 배경, `#333` dashed 테두리
- 해당 슬롯의 EquipmentIcon (COMMON 등급, opacity 0.15)
- 하단 슬롯명 라벨 (9px, #555)
- 클릭 → 보관함을 해당 슬롯으로 필터

**장착된 슬롯:**
- `#1a1a2e` 배경, 등급 색상 solid 테두리
- EquipmentIcon (등급 색상 적용)
- 하단 슬롯명 라벨
- 클릭 → 상세 패널 표시 + 보관함을 해당 슬롯으로 필터

**선택된 슬롯:**
- 테두리 강조 (`#e94560`, box-shadow glow)

| 슬롯 | 최대 장착 수 |
|------|-------------|
| 무기 | 1 |
| 방어구 | 1 |
| 반지 | 2 |
| 목걸이 | 1 |
| 신발 | 1 |
| 장갑 | 1 |
| 모자 | 1 |

#### 장비 상세 패널 (선택 시 표시)

**장착 장비 선택 시:**
- 장비 이름 (등급 색상) + Lv.N
- 스탯: ATK +N, HP +N, DEF +N, CRIT +N%
- 패시브 능력 설명
- 강화석 잔량 표시
- [강화] [해제] 버튼
- S등급: `[S]` 접두사 표시
- **강화 버튼**: 장비 강화석 1개 소모, 강화석 부족 시 비활성
- **해제 버튼**: 장비를 보관함으로 이동

**보관함 장비 선택 시:**
- 장비 이름 (등급 색상) + Lv.N
- 스탯: ATK +N, HP +N, DEF +N, CRIT +N%
- 패시브 능력 설명
- [장착] [{가격}G] 버튼
- S등급: `판매불가` 비활성 버튼

#### 보관함 아이콘 그리드

**보관함 헤더:**
- "보관함 (N)" + "강화석: N" 표시

**필터 탭 바:**
```
[전체] [무기] [방어구] [반지] [목걸이] [신발] [장갑] [모자]
```
- 초기 상태: 필터 미선택 → 아이콘 그리드 미표시 (필터 버튼만 표시)
- 전체: 모든 보관함 장비 표시
- 슬롯별: 해당 슬롯만 필터
- 페이퍼돌 슬롯 클릭 시 자동으로 해당 슬롯 필터 적용

**아이콘 그리드 (필터 선택 시에만 표시):**
- 5열 CSS Grid
- 각 셀: EquipmentIcon (size=32) + 등급 테두리 + 등급 라벨(9px) + 슬롯명(9px)
- S등급: 금색 테두리
- 선택된 아이템: `#e94560` 테두리
- mergeLevel > 0: "+N" 뱃지 표시
- 정렬: 등급 높은 순 → S등급 우선 → 레벨 높은 순
- 빈 보관함: "보관 중인 장비가 없습니다" 표시

#### 판매 가격표

| 등급 | 판매가 |
|------|--------|
| 일반 | 10G |
| 우수 | 30G |
| 희귀 | 100G |
| 에픽 | 300G |
| 전설 | 1000G |
| 신화 | 3000G |

### 2-2. 합성 탭

#### 일괄 합성 버튼
```
[ 일괄 합성 (N건) ]
```
- 합성 가능한 모든 그룹을 한 번에 합성
- 합성 가능 그룹이 0건이면 비활성
- 완료 후 결과 카드 표시 (4초 후 자동 사라짐): 각 결과 장비의 EquipmentIcon + 등급 색상 이름

#### 합성 미리보기 영역
```
┌────────────────────────────────────┐
│ [Icon]+[Icon]+[Icon] = [ResultIcon]│
│ 결과: {결과 등급+슬롯명}           │
│ ATK +N  HP +N                      │
│ 패시브: {패시브 설명}              │
│ [     합성하기     ]               │
└────────────────────────────────────┘
```
- 아이콘 그리드에서 장비 미선택 시: "장비를 선택하면 합성 미리보기가 표시됩니다" placeholder
- 선택 시: 아이콘 다이어그램 (보유=실선, 부족=점선), 결과 등급/스탯/패시브 표시
- 합성 가능 시 "합성하기" 버튼 활성, 불가 시 "N개 더 필요" 텍스트

#### 장비 아이콘 그리드
```
┌──┐┌──┐┌──┐┌──┐┌──┐
│🗡││🗡││🛡││💍││📿│  ← 5열 CSS Grid
│3/3│2/2│1/3│3/3│2/3│  ← 수량 뱃지
└──┘└──┘└──┘└──┘└──┘
```
- 보관함 장비를 **등급+슬롯(+무기종류+mergeLevel)** 기준으로 그룹핑
- 5열 그리드, 각 셀: EquipmentIcon + 등급 라벨 + 슬롯명 + 수량 뱃지
- 합성 가능 그룹: 초록 테두리 glow
- 선택된 그룹: 빨간 강조 테두리
- 클릭 시 상단 미리보기 영역에 해당 그룹의 합성 정보 표시
- 그룹 정렬: 등급 높은 순 → 슬롯명 → mergeLevel 높은 순
- S등급/신화 등급: 그리드에서 제외

## 3. 합성 규칙

### 저등급 (일반~희귀): 3개 합성

| 재료 등급 | 필요 수량 | 결과 등급 |
|-----------|----------|-----------|
| 일반 | 3 | 우수 |
| 우수 | 3 | 희귀 |
| 희귀 | 3 | 에픽 |

### 고등급 (에픽 이상): 2개 mergeLevel 단계 합성

| 재료 | 필요 수량 | 결과 |
|------|----------|------|
| 에픽+0 | 2 | 에픽+1 |
| 에픽+1 | 2 | 에픽+2 |
| 에픽+2 | 2 | 전설+0 |
| 전설+0 | 2 | 전설+1 |
| 전설+1 | 2 | 전설+2 |
| 전설+2 | 2 | 신화 |

- 에픽 이상 장비는 같은 등급 + 같은 mergeLevel 2개를 합성
- mergeLevel이 최대(+2)에 도달하면 다음 등급으로 승격
- 합성 그룹은 mergeLevel별로 분리 표시
- 장비 이름에 mergeLevel > 0이면 "+N" 접미사 표시
- S등급 장비는 합성 재료로 사용 불가

## 4. 데이터 매핑

### 슬롯 탭

```
슬롯 목록    → Player.getEquipmentSlot(slotType)
장착 장비    → EquipmentSlot.equipped[]
보관함 장비  → Player.inventory.filter(e => e.slot === slotType)
장비 스탯    → Equipment.getStats() → { atk, maxHp }
강화석       → Player.resources.equipmentStones
장착 처리    → Player.equipFromInventory(id)
해제 처리    → Player.unequipToInventory(slotType, index)
판매 처리    → Player.sellEquipment(id)
판매 가격    → SELL_PRICES[grade]
```

### 합성 탭

```
합성 후보    → Forge.findMergeCandidates(inventory)
합성 가능여부 → Forge.canMerge(equipments)
합성 실행    → Forge.merge(equipments)
필요 수량    → EquipmentTable.getMergeCount(grade)
다음 등급    → EquipmentTable.getNextGrade(grade)
등급 인덱스  → EquipmentTable.getGradeIndex(grade)
```

## 6. 상호작용 흐름

### 능력치 변화 알림
- 장착/해제/강화 시 능력치 변화량을 화면 중앙 토스트로 표시 (0.7초)
- 증가: 초록색 (+), 감소: 빨간색 (-)
- ATK, HP, DEF, CRIT 변화분만 표시

### 장비 강화
```
페이퍼돌 장착 슬롯 클릭 → 상세 패널 → [강화] → 강화석 1개 소모 → 레벨 +1 → 슬롯 레벨 동기화 → 능력치 변화 토스트 → 자동 저장
```
- 강화 레벨은 슬롯에 귀속 (장비 교체/해제해도 슬롯 레벨 유지)
- 새 장비 장착 시 슬롯 레벨이 자동 적용됨

### 장비 해제/장착
```
페이퍼돌 장착 슬롯 클릭 → 상세 패널 → [해제] → 보관함으로 이동 → 자동 저장
보관함 그리드 아이콘 클릭 → 상세 패널 → [장착] → 빈 슬롯 또는 최약 장비 교체 → 자동 저장
```

### 장비 판매
```
보관함 그리드 아이콘 클릭 → 상세 패널 → [{가격}G] → 골드 획득 → 주간퀘스트(weekly_sell) 진행 → 자동 저장
```

### 장비 합성
```
합성 탭 → 그룹 카드 클릭 → 상세 패널 확인 → [합성하기] 클릭
→ 재료 장비 인벤토리에서 제거 → 결과 장비 인벤토리에 추가 → 자동 저장
```
