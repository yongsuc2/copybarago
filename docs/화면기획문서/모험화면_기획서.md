## 1. 화면 구성 (위 → 아래)

### 1-1. 전투 상단 바
```
[⚙] ────── 턴 {현재턴}/{최대턴} ──────
[스킬1][스킬2][스킬3][스킬4]  ← 세션 스킬 아이콘 목록
```
- 턴 카운터: 현재 턴 / 최대 턴(15) 표시
- 세션 스킬 아이콘: 챕터 진행 중 획득한 스킬을 아이콘으로 표시 (최대 8개)

### 1-2. 전투 영역 (메인)
```
        +37.3K(초록)         히트 2
                           -64.1K(빨강)

  [플레이어 캐릭터]     [몬스터1] [몬스터2]
   ████████░░ HP바       ██░░░░░░ HP바
   282.2K               309.3K

                              [x2] [📊]
```
- **플레이어 캐릭터**: 좌측 배치, 카피바라 SVG 아이콘
- **몬스터**: 우측 배치, 몬스터 종류별 SVG 아이콘
- **HP 바**: 캐릭터 아래 초록색(플레이어) / 빨간색(몬스터) 게이지
- **전투력 표시**: HP 바 아래 숫자 (K 단위 포맷)
- **데미지 표시**: 공격 시 피해량 팝업 (빨강: 피해, 초록: 회복)
- **히트 표시**: 연타 시 히트 카운트 표시
- **분노 게이지**: 플레이어 HP 바 아래 주황→빨강 그라데이션 게이지, 현재값/최대값 표시
- **x2 버튼**: 전투 속도 2배속
- **📊 버튼**: 상세 전투 로그 토글

### 1-3. 플레이어 스탯 바
```
[EXP]     [❤️ HP]                [⚔ 공격력]  [🛡 방어력]
 LV.4   100% 282.2K/282.2K      53.4K       8995
```
- EXP / 레벨 표시
- HP: 퍼센트 + 현재HP/최대HP (K 단위 포맷)
- 공격력: 아이콘 + 수치
- 방어력: 아이콘 + 수치

### 1-4. 챕터 진행 바
```
████████████░░░░░░░░ (일차 기반 진행률)
```
- 현재 일차 / 총 일차 기반 프로그레스 바

### 1-5. 인카운터 영역
```
─── {N}일차 ───

┌─────────────────────────────┐
│ {인카운터 설명 텍스트}         │
│                             │
│ [선택지1]                    │
│ [선택지2]                    │
│ [선택지3]                    │
└─────────────────────────────┘
```
- 일차 표시 (구분선 포함)
- 인카운터 타입에 따른 설명 텍스트
- 선택지 버튼 목록

### 1-6. 하단 정보
```
[3/7]                          [11/12]
[대박 아이콘]  ──[전투 중]──  [중박 아이콘]
 대박                          중박
```
- 대박/중박 룰렛 카운트 (60일 챕터)
- 중앙: 현재 상태 버튼 (전투 중 / 다음 날 등)

## 2. 화면 상태별 분기

**핵심 구조: AdventureStage (상시 스테이지)**

챕터 진행 중에는 `AdventureStage` 컴포넌트가 항상 표시된다. 레이아웃이 상태에 따라 완전히 바뀌지 않고, 스테이지 영역은 고정되며 내부 콘텐츠만 전환된다.

```
adventure-stage (고정 높이 170px, 다크 배경 #0d1b2a)
├── 전투 시: BattleArena 전체 렌더링 (기존 동작 그대로)
└── 비전투 시:
    └── ba-field (기존 레이아웃 재활용)
        ├── ba-character (카피바라, 항상 표시)
        └── stage-encounter-visual (인카운터 이모지, 선택적)
```

**인카운터 비주얼 매핑:**

| EncounterType | 이모지 |
|---------------|--------|
| COMBAT | ⚔️ |
| ANGEL | 😇 |
| DEMON | 😈 |
| CHANCE(상자) | 🎁 |
| CHANCE(샘) | 💧 |
| CHANCE(축복) | ✨ |
| JUNGBAK_ROULETTE | 🎲 |
| DAEBAK_ROULETTE | ⭐ |

### 상태 A: 챕터 미진행
- 다음 챕터 번호, 스태미나 소모량 표시
- "챕터 시작" 버튼
- 보물상자 버튼

### 상태 B: 인카운터 진행 중
- **AdventureStage**: 좌측 카피바라 + 우측 인카운터 이모지 표시
- 챕터 진행바: 챕터 N / 현재일 / 총일 + 스킬 수
- 스탯 바: 현재 스탯 표시
- 인카운터 영역: 일차 구분선 + 타입 라벨(한글) + 선택지 표시
- 포기하기 버튼

### 상태 C: 전투 중
- **AdventureStage**: BattleArena 모드 (플레이어 + 몬스터, HP바, 데미지 팝업)
- 챕터 진행바: 챕터 N / 현재일 / 총일
- 스탯 바: 실시간 HP 반영
- 포기하기 버튼
- 전투 로그: 턴별 기록 업데이트

### 상태 D: 챕터 결과
- **AdventureStage**: 대기 모드 (카피바라만 표시)
- 결과 오버레이: 승리/패배 아이콘, 챕터 번호, 보상 표시
- 메인으로/다음 챕터 시작/보물상자 확인 버튼

### 상태 E: 보스 전투
- 상태 C와 동일하되, 몬스터 영역에 보스 아이콘 (크기 1.4배)
- 상단에 "BOSS" 표시

## 5. 데이터 매핑

### 스탯 바 데이터 소스
```
HP       → BattleUnit.currentHp / BattleUnit.stats.maxHp
공격력    → BattleUnit.stats.atk
방어력    → BattleUnit.stats.def
레벨     → Player.talent.getTotalLevel() 기반
```

### 전투 영역 데이터 소스
```
플레이어 HP바  → playerUnit.currentHp / playerUnit.stats.maxHp
몬스터 HP바    → enemyUnit.currentHp / enemyUnit.stats.maxHp
전투력 수치    → BattleUnit.stats.atk + BattleUnit.stats.maxHp (전투력 공식)
데미지 표시    → BattleLog 엔트리의 damage 값
```

### 전투 로그 → 데미지 팝업 매핑
```
ATTACK, SKILL_DAMAGE → 빨간 데미지 숫자 (-)
CRIT                 → 빨간 데미지 숫자 (크게) + "치명타" 텍스트
LIFESTEAL, HOT_HEAL  → 초록 회복 숫자 (+)
COUNTER              → 노란 데미지 숫자
RAGE_ATTACK          → 주황 데미지 숫자 (크게) + 글로우 효과
```

## 6. 챕터 보물상자 화면

화면 파일: `src/presentation/screens/ChapterTreasureScreen.tsx`
진입: 모험 화면 시작 카드의 "보물상자" 버튼 또는 챕터 결과 화면의 "보물상자 확인" 버튼

### 6-1. 구조 (선형 진행 방식)

챕터별 4개 마일스톤(15일, 25일, 40일, 클리어)이 선형으로 나열된다.
Ch1 15일 → Ch1 25일 → Ch1 40일 → Ch1 클리어 → Ch2 15일 → ... 순서.

탭이나 배너 선택 없이, **아직 수령하지 않은 첫 번째 마일스톤**을 자동 표시한다.
수령 버튼 클릭 시 보상 지급 후 다음 마일스톤으로 자동 전환.

```
┌─────────────────────────────────┐
│       챕터 보물상자               │  ← 타이틀 배너
├─────────────────────────────────┤
│  보상 진행도                      │
│  ████████░░ 5/12                │  ← 수령완료 / 전체
├─────────────────────────────────┤
│  챕터 2 — 25일                   │  ← 현재 마일스톤
│  최고 기록: 18일                  │
│                                 │
│  ┌──────┐ ┌──────┐              │
│  │1000G │ │ 60💎 │ ...          │  ← 보상 그리드
│  └──────┘ └──────┘              │
│                                 │
│  챕터 2에서 25일 생존 후 수령 가능 │  ← 조건 텍스트
│  [ 수령 ] / [ 조건 미달성 ]      │  ← 버튼
├─────────────────────────────────┤
│  ← 뒤로가기                      │
└─────────────────────────────────┘
```

### 6-2. 마일스톤 상태

| 상태 | 표시 |
|------|------|
| claimable | "수령" 버튼 활성화 |
| locked | "조건 미달성" 버튼 비활성화 |
| 전부 claimed | "모든 보상을 수령했습니다" 메시지 |

### 6-3. 데이터 소스

```
전체 마일스톤   → ChapterTreasureTable.getAvailableChapterIds() + getMilestonesForChapter()
수령 여부       → Player.claimedMilestones (Set<string>)
최고 기록       → Player.bestSurvivalDays (Map<number, number>)
보상 지급       → GameManager.claimChapterTreasure(milestoneId)
```
