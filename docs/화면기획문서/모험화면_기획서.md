## 1. 화면 구성 (위 → 아래)

### 1-1. 전투 상단 바
```
[⚙] ────── 턴 {현재턴}/{최대턴} ──────
[스킬1][스킬2][스킬3][스킬4]  ← 세션 스킬 아이콘 목록
```
- 턴 카운터: 현재 턴 / 최대 턴(15) 표시
- 세션 스킬 아이콘: 챕터 진행 중 획득한 스킬을 아이콘으로 표시 (최대 8개)

### 1-2. 전투 영역 (메인)
```
        +37.3K(초록)         히트 2
                           -64.1K(빨강)

  [플레이어 캐릭터]     [몬스터1] [몬스터2]
   ████████░░ HP바       ██░░░░░░ HP바
   282.2K               309.3K

                              [x2] [📊]
```
- **플레이어 캐릭터**: 좌측 배치, 카피바라 SVG 아이콘
- **몬스터**: 우측 배치, 몬스터 종류별 SVG 아이콘
- **HP 바**: 캐릭터 아래 초록색(플레이어) / 빨간색(몬스터) 게이지
- **전투력 표시**: HP 바 아래 숫자 (K 단위 포맷)
- **데미지 표시**: 공격 시 피해량 팝업 (빨강: 피해, 초록: 회복)
- **히트 표시**: 연타 시 히트 카운트 표시
- **x2 버튼**: 전투 속도 2배속
- **📊 버튼**: 상세 전투 로그 토글

### 1-3. 플레이어 스탯 바
```
[EXP]     [❤️ HP]                [⚔ 공격력]  [🛡 방어력]
 LV.4   100% 282.2K/282.2K      53.4K       8995
```
- EXP / 레벨 표시
- HP: 퍼센트 + 현재HP/최대HP (K 단위 포맷)
- 공격력: 아이콘 + 수치
- 방어력: 아이콘 + 수치

### 1-4. 챕터 진행 바
```
████████████░░░░░░░░ (일차 기반 진행률)
```
- 현재 일차 / 총 일차 기반 프로그레스 바

### 1-5. 인카운터 영역
```
─── {N}일차 ───

┌─────────────────────────────┐
│ {인카운터 설명 텍스트}         │
│                             │
│ [선택지1]                    │
│ [선택지2]                    │
│ [선택지3]                    │
└─────────────────────────────┘
```
- 일차 표시 (구분선 포함)
- 인카운터 타입에 따른 설명 텍스트
- 선택지 버튼 목록

### 1-6. 하단 정보
```
[3/7]                          [11/12]
[대박 아이콘]  ──[전투 중]──  [중박 아이콘]
 대박                          중박
```
- 대박/중박 룰렛 카운트 (60일 챕터)
- 중앙: 현재 상태 버튼 (전투 중 / 다음 날 등)

## 2. 화면 상태별 분기

### 상태 A: 챕터 미진행
- 다음 챕터 번호, 스태미나 소모량 표시
- "챕터 시작" 버튼
- 이전 결과 (클리어/실패) 표시

### 상태 B: 인카운터 진행 중
- 전투 영역: 플레이어 캐릭터 대기 상태 (몬스터 없음)
- 스탯 바: 현재 스탯 표시
- 인카운터 영역: 선택지 표시
- 전투 로그: 이전 턴 기록

### 상태 C: 전투 중
- 전투 영역: 플레이어 + 몬스터 표시, HP바 표시
- 데미지 팝업 표시
- 스탯 바: 실시간 HP 반영
- 하단: "전투 중" 상태 표시
- 전투 로그: 턴별 기록 업데이트

### 상태 D: 전투 결과
- 승리: 승리 연출 → 다음 인카운터로 자동 진행
- 패배: 패배 연출 → 챕터 종료

### 상태 E: 보스 전투
- 상태 C와 동일하되, 몬스터 영역에 보스 아이콘 (크기 1.5배)
- 상단에 "BOSS" 표시

## 4. 캐릭터 표현

SVG 기반 단순 아이콘으로 표현 (이미지 파일 아님)

### 플레이어 (카피바라)
- 둥근 갈색 몸체, 작은 귀, 점 눈
- 전투 시 공격 애니메이션: 좌→우 이동 후 복귀 (CSS transition)

### 몬스터 종류
| 타입 | 색상 | 형태 |
|------|------|------|
| 슬라임 | 초록 | 물방울 형태 |
| 골렘 | 회색 | 사각 바위 형태 |
| 드래곤 | 빨강 | 삼각 날개 형태 |
| 유령 | 보라 | 반투명 둥근 형태 |
| 보스 | 검정+빨강 | 일반 몬스터 1.5배 크기 |

## 5. 데이터 매핑

### 스탯 바 데이터 소스
```
HP       → BattleUnit.currentHp / BattleUnit.stats.maxHp
공격력    → BattleUnit.stats.atk
방어력    → BattleUnit.stats.def
레벨     → Player.talent.getTotalLevel() 기반
```

### 전투 영역 데이터 소스
```
플레이어 HP바  → playerUnit.currentHp / playerUnit.stats.maxHp
몬스터 HP바    → enemyUnit.currentHp / enemyUnit.stats.maxHp
전투력 수치    → BattleUnit.stats.atk + BattleUnit.stats.maxHp (전투력 공식)
데미지 표시    → BattleLog 엔트리의 damage 값
```

### 전투 로그 → 데미지 팝업 매핑
```
ATTACK, SKILL_DAMAGE → 빨간 데미지 숫자 (-)
CRIT                 → 빨간 데미지 숫자 (크게) + "치명타" 텍스트
LIFESTEAL, HOT_HEAL  → 초록 회복 숫자 (+)
COUNTER              → 노란 데미지 숫자
```

## 6. 구현 우선순위

### Phase 1 (핵심)
1. 플레이어 스탯 바 (HP, 공격력, 방어력)
2. 전투 영역 (플레이어/몬스터 SVG + HP바)
3. 턴 카운터
4. 챕터 진행 바 + 일차 표시

### Phase 2 (연출)
5. 데미지 팝업 애니메이션
6. 캐릭터 공격 애니메이션
7. 전투 속도 조절 (x2)

### Phase 3 (부가)
8. 세션 스킬 아이콘 목록
9. 대박/중박 카운트 표시
10. 상세 전투 로그 패널
