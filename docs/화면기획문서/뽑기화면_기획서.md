# 뽑기화면 기획서

## 화면 구성 (위 → 아래)

### 1-1. 탭 바 (고정 높이)
```
[장비] [펫] [보석]
```
- 3개 탭 전환
- 현재 활성 상자 타입에 따라 하단 정보 갱신

### 1-2. 쇼케이스 영역 (유연 높이 — 남은 공간 채움)
```
┌────────────────────────────────┐
│                                │
│         ┌────────┐             │
│         │ 상자   │             │
│         │ 아이콘  │             │
│         └────────┘             │
│        {상자 타입 이름}         │
│                                │
└────────────────────────────────┘
```
- 다크 배경 (Background 계열)
- 중앙에 상자 아이콘 (탭별 색상 변경)
  - 장비: Legendary 색상
  - 펫: Heal 색상
  - 보석: Gems 색상
- 아이콘 아래 상자 타입 이름 (볼드)
- 상하 스페이서로 수직 중앙 정렬

### 1-3. 정보 카드 (고정 높이)
```
┌────────────────────────────────┐
│ 보석: {보유량}                  │
│   1회: {비용}    10회: {비용}   │
│ 천장: {현재}/{임계값}           │
│ ████████░░░░░░ (진행도 바)     │
└────────────────────────────────┘
```
- Card 배경
- 보석 보유량 (Gems 색상)
- 1회/10회 비용 (가로 나란히)
- 천장 카운터 텍스트
- 천장 프로그레스 바 (Legendary 색상)

### 1-4. 뽑기 버튼 행 (고정 높이 — UISize.NormalButton 기준)
```
[ 1회 ({비용}) ]  [ 10회 ({비용}) ]
```
- 1회 뽑기 버튼 (Primary 색상)
- 10회 뽑기 버튼 (Legendary 색상)
- 보석 부족 시 비활성 처리

## 뽑기 결과 팝업 (GachaRewardPopup)

### 구조
- 팝업 오버레이 위에 카드 형태로 표시
- 상단: 제목 ("뽑기 결과", 10회일 경우 횟수 표시)
- 중앙: 결과 그리드 (스크롤 가능, RectMask2D)
- 하단: 버튼 행

### 결과 그리드
- 5열 그리드 레이아웃
- 각 셀 구성:
  - 장비: 등급 색상 배경 + 장비 아이콘 + S 마크(해당 시) + 이름(등급 색상) + 등급 텍스트(PITY 표시)
  - 재화: 기본 배경 + 재화 아이콘 + 타입명 + 수량
- 결과가 많으면 스크롤 가능

### 하단 버튼
- "닫기" 버튼 (Secondary 색상) — 팝업 닫기
- "한번 더" 버튼 (Primary 색상) — 팝업 닫고 동일 유형 뽑기 재실행
- 버튼 크기: UISize의 NormalButton 기준 적용

---

## 상호작용 흐름

1. 탭 선택 → 해당 상자의 비용/천장 정보 갱신
2. 뽑기 버튼 클릭 → 보석 차감 + 결과 생성 + 인벤토리 추가 + 저장
3. 결과 팝업 표시 → 장비/재화 그리드로 확인
4. "닫기" → 팝업 닫힘, 화면 갱신됨
5. "한번 더" → 팝업 닫힘 + 즉시 다음 뽑기 실행 + 새 결과 팝업

## 데이터 매핑

```
보유 보석       → Player.resources.Gems
상자 시스템     → Game.EquipmentChestSystem (현재 장비만)
1회 비용        → chest.GetCostPerPull()
10회 비용       → chest.GetPull10Cost()
천장 카운트     → chest.PityCount
천장 임계값     → chest.GetPityThreshold()
1회 뽑기        → Game.PullGacha()
10회 뽑기       → Game.PullGacha10()
```
