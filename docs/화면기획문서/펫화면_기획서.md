# 펫 화면 기획서

---

## 1. 화면 구성 (위 → 아래)

### 1-1. 화면 헤더
```
펫
```

### 1-2. 재화 정보 카드
```
┌────────────────────────────────┐
│ 펫 알                   {수량} │
│ 펫 먹이                 {수량} │
│                                │
│ [       알 부화       ]        │
└────────────────────────────────┘
```
- 펫 알 0개이면 부화 버튼 비활성

### 1-3. 빈 상태
```
보유한 펫이 없습니다
```
- 펫이 0마리일 때만 표시

### 1-4. 펫 카드 목록
```
┌────────────────────────────────┐  ← 출전 중 펫: 빨간 테두리(#e94560)
│ [PetIcon] 펫이름               │
│  44x44    [{티어}] {등급}  N레벨│
│           공격력 +N    체력 +N  │
│           EXP N/N  먹이 N개→레벨업│
│           ████░░░░ (경험치 바)  │
│────────────────────────────────│
│ 특수능력  {능력 설명}           │
│────────────────────────────────│
│ [출전/출전 중]  [먹이주기]      │
└────────────────────────────────┘
```

---

## 2. 펫 카드 상세

### 2-1. 헤더 영역 (PetIcon + 정보)
- **PetIcon**: 펫별 고유 SVG 아이콘 (18종), 티어별 배경 색상
  - S티어: #ffd740 (금색)
  - A티어: #ce93d8 (보라)
  - B티어: #81c784 (초록)
- **이름 + 등급**: 펫 이름, [티어] 등급 표시
- **레벨**: 현재 레벨
- **스탯 보너스**: 공격력/체력 증가량 (레벨 기반)
- **경험치 바**: 현재 EXP / 다음 레벨 필요 EXP + 프로그레스 바
- **필요 먹이 수**: "먹이 N개 → 레벨업" (남은 EXP를 먹이당 EXP로 나눈 올림값)

### 2-2. 특수능력 영역
- "특수능력" 라벨 (금색) + 능력 설명
- 능력 설명은 PetTable에서 동적 생성 (등급에 따른 수치 반영)
- 능력이 없는 펫은 영역 비표시

### 2-3. 버튼 영역
- **출전**: 해당 펫을 활성 펫으로 설정 (비출전 상태에서만 표시)
- **출전 중**: 현재 활성 펫 (비활성 상태 표시)
- **먹이주기**: 펫 먹이 1개 소모, 경험치 획득 → 레벨업 가능. 먹이 0개이면 비활성

---

## 3. 경험치/레벨 시스템

- 먹이당 경험치: 데이터 테이블 참조 (Pet 엔티티)
- 다음 레벨 필요 경험치: 데이터 테이블 참조 (레벨에 따라 증가)
- 레벨당 스탯 보너스: 데이터 테이블 참조 (Pet 엔티티)

## 4. 등급 체계

| 등급 | 영문 |
|------|------|
| 일반 | COMMON |
| 희귀 | RARE |
| 에픽 | EPIC |
| 전설 | LEGENDARY |
| 불멸 | IMMORTAL |

## 5. 데이터 매핑

```
펫 목록       → Player.ownedPets
활성 펫       → Player.activePet
펫 알 수량    → Player.resources.get(PET_EGG)
펫 먹이 수량  → Player.resources.get(PET_FOOD)
펫 아이콘     → PetIcon 컴포넌트 (petId, tier)
펫 스탯       → Pet.getGlobalBonus()
경험치 정보   → Pet.exp, Pet.getExpToNextLevel()
특수능력 설명 → PetTable.getAbilityDescription(templateId, grade)
부화 처리     → PetManager.hatchEgg(rng)
먹이 처리     → Pet.feed(amount)
출전 처리     → Player.setActivePet(pet)
```

---

## 6. 상호작용 흐름

### 알 부화
```
[알 부화] 클릭 → 펫 알 1개 소모 → PetManager.hatchEgg() → 펫 목록에 추가
→ 활성 펫 없으면 자동 출전 → 자동 저장
```

### 먹이주기
```
[먹이주기] 클릭 → 펫 먹이 1개 소모 → Pet.feed(1) → 경험치 획득
→ 경험치 충분 시 레벨업 (스탯 증가) → 자동 저장
```

### 출전
```
[출전] 클릭 → Player.setActivePet(pet) → 기존 활성 펫 해제 → 자동 저장
```
