# 카피바라고! 역기획서 - 스킬 시스템

## 개요

스킬은 **챕터 진행 중 획득**하는 일시적 능력이다.
챕터 종료 시 스킬은 초기화되며, 매 챕터마다 새로 조합한다.

스킬은 **액티브 스킬**과 **패시브 스킬**로 분류되며, 별도 구조로 관리된다.
액티브 스킬은 계층 구조(상위→하위→최하위)를 가지며, 스킬 연쇄 발동이 가능하다.

수치 데이터: `ActiveSkillTierData.ts`, `PassiveSkillTierData.ts` 참조
스킬 정의: `ActiveSkillRegistry.ts`, `PassiveSkillRegistry.ts` 참조
전투 계수: `BattleDataTable.ts` 참조

## 스킬 계층 구조

액티브 스킬은 4단계 계층을 가진다. **상위→하위→최하위** 방향의 발동만 허용하여 무한루프를 원천 차단한다.

| 계층 | 설명 | 트리거 가능 대상 |
|------|------|----------------|
| 내장 (BUILTIN) | 항상 장착, 획득/제거 불가 | LOWER, LOWEST |
| 상위 (UPPER) | 플레이어가 인카운터에서 획득 | LOWER, LOWEST |
| 하위 (LOWER) | 상위에 의해 발동, 최하위 발동 가능 | LOWEST |
| 최하위 (LOWEST) | 터미널 실행, 추가 연쇄 불가 | 없음 |

최대 체인 깊이 = 3. 런타임 안전장치로 `MAX_SKILL_CHAIN_DEPTH` 초과 시 중단.

### 연쇄 예시

```
번개 수리검 (UPPER) → 수리검 소환 (LOWER) → 번개 소환 (LOWEST)
분노 공격 번개 (UPPER) → 번개 소환 (LOWEST)
일반 공격 (BUILTIN) → 분노 축적 (LOWEST)
```

## 스킬 티어 시스템

모든 스킬 패밀리는 4단계 티어를 가진다. 동일 패밀리의 스킬을 이미 보유한 상태에서 다시 획득하면 다음 티어로 **교체**된다 (슬롯 1개 유지).

| 티어 | 등급 | 설명 |
|------|------|------|
| 1 | 일반(NORMAL) | 최초 획득 시 기본 등급 |
| 2 | 전설(LEGENDARY) | 1회 업그레이드 |
| 3 | 신화(MYTHIC) | 2회 업그레이드 |
| 4 | 불멸(IMMORTAL) | 최대 티어 |

- 스킬 이름에 티어 접미사 표시: tier 1 = 이름만, tier 2+ = `II`, `III`, `IV`
- 티어별 수치는 데이터 테이블 참조
- 티어 스케일링 비율: T1 기준 **1 : 2.3 : 4 : 6** (증가분이 티어마다 커지는 가속 곡선)
  - count(소환 횟수): 1 → 2 → 4 → 6 (정수 반올림)
  - duration: 별도 스케일링 (가속 비율 미적용)
  - 확률 값은 1.0(100%) 캡 — 캡에 걸리는 스킬은 T1을 역산하여 T4=1.0에 도달하도록 조정
- 예외: `angel_power`, `demon_power`는 단일 티어 4(불멸) — 대박 룰렛에서만 획득

### 인카운터별 티어 제한

| 인카운터 | 제공 티어 범위 |
|---------|--------------|
| 천사(중박) | tier 1~2 |
| 악마(대박) | 전체 |
| 중박 룰렛 | 전체 |
| 대박 룰렛 | tier 3 (신화급) + angel_power/demon_power |
| 금상자 (엘리트/보스 승리) | tier 3 (신화급) |

## 발동 조건 시스템

액티브 스킬의 발동 조건은 3개 조건의 AND 조합이다. 모두 만족해야 발동.

### 조건 1 (When)

| 종류 | 파라미터 | 예시 |
|------|---------|------|
| `EVERY_N_TURNS` | interval | 2턴마다 |
| `ON_SKILL_ACTIVATION` | skillId | '일반 공격' 스킬 발동시 |

### 조건 2 (Probability)

확률 (0.0~1.0). 기본값 1.0.

### 조건 3 (Special)

| 종류 | 파라미터 | 설명 |
|------|---------|------|
| `NONE` | — | 조건 없음 |
| `RAGE_FULL` | — | 분노 게이지 100일 때 |
| `HP_BELOW` | threshold | 체력 N% 이하 |
| `HP_ABOVE` | threshold | 체력 N% 이상 |
| `HP_BELOW_ONCE` | threshold | 전투당 체력 N% 이하일 때 단 한번 |

## 공격 타입

방어력은 **퍼센트 감소 공식** 적용: `damage = ATK × 계수 × (k / (k + DEF))`

| 타입 | 계산 방식 | DEF 상수 | 크리티컬 | 용도 |
|------|----------|----------|---------|------|
| 물리 (PHYSICAL) | ATK × 스킬계수 × (k / (k + DEF)) | `BattleDataTable.damage.defenseConstant` | 적용 | 일반 공격, 검기 |
| 마법 (MAGIC) | ATK × 마법계수 × 스킬계수 × (k / (k + DEF)) | `BattleDataTable.damage.magicDefenseConstant` | 미적용 | 번개, 광창, 화염 |
| 고정 (FIXED) | floor(ATK × 스킬계수) | DEF 무시 | 미적용 | 독 (DoT) |

마법계수(기본값 `BattleDataTable.damage.baseMagicCoefficient`)는 magic_mastery 패시브로 강화 가능. 크리티컬은 물리 공격에만 적용된다.

### 광역 공격 (AoE)

ATTACK 효과에 `isAoe: true`가 설정된 스킬은 살아있는 모든 적에게 데미지를 적용한다. 현재 검기 소환(sword_aura_summon)이 광역 공격을 사용한다.

## DoT (지속 피해) 메카닉

### 즉발 피해 vs 지속 피해

DoT 스킬(독 주입, 화염 소환)은 **2단계 데미지**를 가진다:

| 단계 | 로그 타입 | 딜 그래프 표시 | 설명 |
|------|-----------|---------------|------|
| 즉발 피해 | `SKILL_DAMAGE` | 스킬 이름 (예: "독 주입") | 스킬 발동 시 즉시 적용되는 고정/마법 피해 |
| 지속 피해 | `DOT_DAMAGE` | "독 피해" (통합) | 매 턴 종료 시 적용되는 상태효과 피해 |

### DoT 틱 타이밍

DoT 피해는 **턴 종료 시에만** 적용된다. 반격/분노 공격 중에는 틱하지 않는다.

```
executeTurn()
├── 플레이어 공격 (일반/스킬/반격/분노 포함)
├── 적 공격 (일반/스킬/반격/분노 포함)
├── ★ 상태효과 틱 (DoT/HoT/버프/디버프 — 여기서만)
└── 승패 판정
```

### DoT 스킬 목록

| 스킬 | 공격 타입 | DoT 타입 | 티어별 수치 |
|------|----------|----------|------------|
| 독 주입 (poison_inject) | FIXED | POISON | `ActiveSkillTierData` 참조 |
| 화염 소환 (flame_summon) | MAGIC | BURN | `ActiveSkillTierData` 참조 |

## 스킬 효과 타입

### 액티브 스킬 효과

| 효과 | 설명 |
|------|------|
| `ATTACK` | 물리/마법/고정 공격. attackType, coefficient 명시 |
| `TRIGGER_SKILL` | 다른 스킬 발동. targetSkillId, count 명시 |
| `INJECT_EFFECT` | 장착 시 타겟 스킬에 효과 추가 (1회). targetSkillId, injectedEffects 명시 |
| `HEAL_HP` | 최대체력 비례(%) HP 회복. amount 명시 |
| `ADD_RAGE` | 분노 게이지 추가. amount 명시 |
| `CONSUME_RAGE` | 분노 게이지 소모. amount 명시 |
| `DEBUFF` | 적에게 디버프. stat, reduction, duration 명시 |

### [스킬에 효과 추가] (INJECT_EFFECT) 메카닉

상위 스킬이 하위 스킬의 동작을 동적으로 수정하는 핵심 메카닉.

1. 스킬 장착 시 `INJECT_EFFECT` 효과를 처리
2. 타겟 스킬의 런타임 효과 목록에 주입된 효과를 추가
3. 원본 스킬 정의는 불변, BattleUnit의 런타임 복사본만 수정

예시: 번개 수리검 장착 → 수리검 소환에 "확률로 번개 소환 발동" 효과 추가

## 패시브 스킬

액티브 스킬과 별도 인터페이스로 관리.

| PassiveType | 적용 시점 | 설명 |
|-------------|----------|------|
| `STAT_MODIFIER` | 전투 시작 / 세션 | 스탯(ATK/DEF/CRIT/HP/RAGE_POWER/MAGIC_COEFFICIENT) 수정 |
| `COUNTER` | 피격 시 | 확률로 일반 공격 반격 (triggerChance 기반) |
| `LIFESTEAL` | 데미지 후 | 가한 피해의 일부 HP 회복 |
| `SHIELD_ON_START` | 전투 시작 | 최대 HP 비례 방어막 생성 |
| `REVIVE` | 사망 시 | HP 일부로 부활 (1회) |
| `REGEN` | 턴 종료 | 매턴 최대체력 비례(%) 회복 |
| `MULTI_HIT` | 일반 공격 후 | 확률로 일반 공격 + UPPER 스킬 재발동 (연타 재발동 없음) |
| `SKILL_MODIFIER` | 스킬 실행 시 | 특정 태그를 가진 스킬의 데미지 증가 |

### SKILL_MODIFIER 상세

특정 속성(태그)의 스킬 데미지를 증가시키는 패시브. 장착 시 해당 태그의 스킬이 발동될 때 최종 데미지에 배율을 적용한다.

**적용 흐름:**
1. 전투 시작 시 패시브 장착 → 태그별 데미지 보너스 저장
2. 스킬 실행 시 해당 스킬의 태그 확인
3. 매칭되는 보너스가 있으면 최종 데미지에 `(1 + 보너스)` 배율 적용

**데미지 공식:**
```
기본 데미지 = ATK × 계수 × (k / (k + DEF)) × 크리티컬(물리만)
최종 데미지 = 기본 데미지 × (1 + 태그 보너스)
```

**현재 스킬 목록:**

| 스킬 | 대상 태그 | 계승 시너지 | 티어별 수치 |
|------|----------|------------|------------|
| 번개 마스터리 (lightning_mastery) | LIGHTNING | 유령 | `PassiveSkillTierData` 참조 |
| 수리검 마스터리 (shuriken_mastery) | SHURIKEN | 유협 | `PassiveSkillTierData` 참조 |
| 광창 마스터리 (lance_mastery) | LANCE | 기사 | `PassiveSkillTierData` 참조 |

**대상 태그별 영향 받는 액티브 스킬:**

| 태그 | 영향 받는 스킬 |
|------|--------------|
| LIGHTNING | 번개 소환, 번개 수리검, 번개 강타, 분노 공격 번개 |
| SHURIKEN | 수리검 소환, 번개 수리검, 분노 수리검, 회복 수리검 |
| LANCE | 광창 소환, 광창 강타, 분노 공격 광창 |

### 세션 HP 패시브 (STAT_MODIFIER + HP)

체력 강화(hp_fortify) 패시브는 세션 레벨에서도 적용된다. 챕터 중 인카운터(악마 계약 등)의 HP 기반 계산이 증가된 maxHp를 참조하도록 보장.

- `Chapter.recalcSessionMaxHp()`: baseSessionMaxHp에서 HP STAT_MODIFIER를 재적용
- HP 비율 보존: 50/100 → 55/110 (currentHp도 비례 스케일링)
- `Chapter.getBattlePassiveSkills()`: HP STAT_MODIFIER를 제외하여 BattleUnit에서 이중 적용 방지
- 전투 시작 시 sessionMaxHp를 BattleUnit stats에 직접 전달

### 마법 마스터리 (STAT_MODIFIER + MAGIC_COEFFICIENT)

마법 마스터리(magic_mastery) 패시브는 마법 계수를 증가시킨다. BattleUnit 생성 시 적용되며, 모든 마법(MAGIC) 공격의 기본 데미지에 영향.

## 스킬 분류

### 내장 스킬

모든 전투에서 자동 장착되는 기본 스킬. `ActiveSkillRegistry` 참조.

- **일반 공격**: 매턴 물리 공격 + 분노 축적 발동
- **분노 공격**: 분노 게이지 100일 때 물리 공격 + 분노 소모

### 상위 스킬 (획득 대상)

인카운터에서 획득하는 스킬. 하위/최하위 스킬을 발동하거나 효과를 주입한다.

수리검 계열, 번개 계열, 광창 계열, 검기 계열, 분노 계열, 독 계열, 화염 계열, 디버프 계열.
각 스킬의 정확한 정의와 티어별 수치는 `ActiveSkillRegistry`, `ActiveSkillTierData` 참조.

### 하위/최하위 스킬

상위 스킬에 의해 발동되는 실행 스킬. 플레이어가 직접 획득하지 않는다.

하위(LOWER): 수리검 소환 — `INJECT_EFFECT`의 핵심 대상, 최하위 연쇄 가능.
최하위(LOWEST): 번개 소환, 광창 소환, 검기 소환, 화염 소환, 독 주입, HP 회복, 분노 축적 — 터미널 실행.

### 패시브 스킬

인카운터에서 획득하는 스탯 수정/반응형 효과.
흡혈, 재생, 반격, 방어막, 연타(일반 공격 전용), 치명타 마스터리, 분노 마스터리, 번개 마스터리, 수리검 마스터리, 광창 마스터리, 공격 강화, 체력 강화, 마법 마스터리, 부활, 천사의 힘 등.
(중복 패시브 병합: ATK%/DEF%/CRIT 그룹당 각 1개만 유지)
각 스킬의 정확한 정의와 티어별 수치는 `PassiveSkillRegistry`, `PassiveSkillTierData` 참조.

## 스킬 특징 (기획/개발 참고)

각 스킬의 기획 의도와 특성을 정리한 참고 자료. 게임 내 UI에는 표시되지 않으며, 코드의 `traits` 필드와 동기화된다.

### 상위 액티브 스킬

| 스킬 | 특징 |
|------|------|
| 번개 수리검 | 수리검+번개 복합 / 확률 기반 추가 피해 |
| 분노 수리검 | 수리검+분노 복합 / 분노 게이지 빠른 축적 |
| 회복 수리검 | 수리검+회복 복합 / 공격과 회복 동시 |
| 번개 강타 | 일반 공격 연계 / 마법 공격 (방어 관통 높음) |
| 광창 강타 | 매턴 자동 발동 / 마법 공격 (방어 관통 높음) |
| 검기 강타 | 일반 공격 연계 / 광역 물리 공격 / 단일 계수 낮음 |
| 분노 공격 번개 | 분노 공격 연계 / 마법 공격 (방어 관통 높음) |
| 분노 공격 광창 | 분노 공격 연계 / 마법 공격 (방어 관통 높음) |
| 분노 공격 화염 | 분노 공격 연계 / 도트 데미지 |
| 분노 게이지 증가 | 일반 공격 연계 / 분노 게이지 빠른 축적 |
| 지독한 검 | 일반 공격 연계 / 고정 피해 (방어 무시) |
| 폭군의 일격 | 일반 공격 연계 / 물리 추가 타격 |
| 축소 마법 | 매턴 자동 발동 / 적 약화 디버프 |
| 악마의 힘 | 일반 공격 연계 / 마법 추가 타격 |

### 패시브 스킬

| 스킬 | 특징 |
|------|------|
| 흡혈 | 물리 공격에 효과적 / 지속적 체력 회복 |
| 재생 | 매턴 자동 회복 / 전투 지구력 향상 |
| 반격 | 피격 시 발동 / 물리 반격 |

## 계층 검증 규칙

데이터 로드 시 자동 검증 (`SkillValidator`):

1. LOWEST 스킬은 `TRIGGER_SKILL` 효과를 가질 수 없음
2. LOWER 스킬의 `TRIGGER_SKILL`은 LOWEST만 대상 가능
3. UPPER 스킬의 `TRIGGER_SKILL`은 LOWER 또는 LOWEST만 대상 가능
4. `INJECT_EFFECT`로 주입된 `TRIGGER_SKILL`도 계층 규칙 준수 확인
5. 자기 참조 금지 (직접 또는 주입을 통한 간접)
